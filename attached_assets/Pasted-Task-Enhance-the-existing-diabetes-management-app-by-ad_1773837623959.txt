Task:
Enhance the existing diabetes management app by adding smarter insulin suggestions, trend-based alerts, improved logging, and a Guardian View system. Do not rebuild the app—extend current functionality.

⸻

1. Trend-Based Glucose Alerts (Enhance Existing Logic)
	•	Add logic to detect glucose trends:
	•	Trending Up
	•	Notify user
	•	Suggest insulin (include estimated amount based on formula)
	•	Trending Down
	•	Notify user
	•	Suggest fast-acting carbs (juice/snack)
	•	Add smart alert messaging:
	•	“You are trending low, consider drinking juice.”
	•	“You are trending high, insulin may be needed.”
	•	Detect abnormal jumps (example: 129 → 255 quickly):
	•	Prompt user:
“This reading seems unusual. Please confirm with a fingerstick.”

⸻

2. Insulin Suggestion (Improve Existing Feature)
	•	Use this formula for ALL insulin suggestions:
	•	Carb Dose = Carbs ÷ ICR
	•	Correction Dose = (Current BG − Target BG) ÷ Correction Factor
	•	Total Dose = Carb Dose + Correction Dose
	•	Apply this when:
	•	Logging food
	•	Taking a picture of food
	•	Output should:
	•	Show insulin estimate
	•	Explain suggestion in simple language

5. Guardian View (New Feature)

Add a Guardian Access Mode for parents/sitters.

Setup (Main Account):
	•	Add section in dashboard: “Guardian View Setup”
	•	Allow user to:
	•	Create username + password
	•	Toggle what guardian can access

Login Screen:
	•	Add option: “Guardian View Login”

Guardian Permissions:
	•	CAN:
	•	View glucose data
	•	View trends
	•	View logs (food + insulin)
	•	Receive alerts
	•	Take food pictures → get insulin suggestion
	•	Log insulin and food
	•	CANNOT:
	•	Change medical settings (ICR, correction factor, etc.)
